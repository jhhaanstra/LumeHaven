# Configuration Reference

LumeHaven is configured via a YAML file (`config.yml`). This document explains all available configuration options with examples.

## Configuration File Structure

```yaml
# Required: GHS connection settings
ghs:
  game_code: "YOUR_GAME_CODE_UUID"
  sqlite_db: "path/to/ghs.sqlite"
  url: "http://localhost:12345"
  interval_ms: 1000

# Optional: Start game monitoring on boot
start_on_boot: false

# Required: Default scene to use on startup
main_scene: "tavern"

# Required: List of lamp configurations
lamps:
  - type: "yeelight"
    id: "living-room-lamp"
    ip: "192.168.0.42"

# Optional: Event-to-effect mappings
effects:
  - event: "fire_element_active"
    effect: "pulse"
    rgb: [255, 0, 0]

# Optional: Scene definitions
scenes:
  - name: "tavern"
    colors:
      - [255, 140, 0]
      - [255, 200, 100]
```

## Configuration Sections

### 1. GHS Settings (`ghs`)

Configures the connection to the Gloomhaven Secretariat server.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `game_code` | string | Yes | - | UUID generated by GHS server. Found in GHS web UI under Server Connection. |
| `sqlite_db` | string | Yes | - | Path to the GHS SQLite database file. Typically `ghs/ghs.sqlite` |
| `url` | string | Yes | - | URL of the GHS server. Default is `http://localhost:12345` |
| `interval_ms` | integer | Yes | - | How often (in milliseconds) to check for game state changes. Recommended: 1000 (1 second) |

**Example:**
```yaml
ghs:
  game_code: "74986287-7208-4719-aba3-5fe464f7f713"
  sqlite_db: "ghs/ghs.sqlite"
  url: "http://localhost:12345"
  interval_ms: 1000
```

### 2. Startup Settings (`start_on_boot`)

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `start_on_boot` | boolean | No | `false` | If `true`, LumeHaven will automatically start monitoring the game on startup. If `false`, you must manually start it via the API (`POST /start`) |

**Example:**
```yaml
start_on_boot: true
```

### 3. Main Scene (`main_scene`)

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `main_scene` | string | Yes | - | Name of the scene to use when LumeHaven starts or when no specific scene is active. Must match a scene name defined in the `scenes` section |

**Example:**
```yaml
main_scene: "tavern"
```

### 4. Lamp Configurations (`lamps`)

List of lamps that LumeHaven will control. Each lamp is an object with the following fields:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Lamp integration type. Must match a registered lamp provider (e.g., `yeelight`, `logging_lamp`) |
| `id` | string | Yes | Unique identifier for this lamp within LumeHaven |
| `ip` | string | Yes | IP address or hostname of the lamp on your network |
| Additional fields | varies | No | Integration-specific configuration (see below) |

**Multiple Lamps Example:**
```yaml
lamps:
  - type: "yeelight"
    id: "living-room-main"
    ip: "192.168.0.42"
  - type: "yeelight"
    id: "living-room-accent"
    ip: "192.168.0.43"
  - type: "logging_lamp"
    id: "debug-lamp"
```

#### Built-in Lamp Types

##### Yeelight

Controls Yeelight smart bulbs using the [yeelight Python library](https://yeelight.readthedocs.io/).

**Required Fields:**
- `type: "yeelight"`
- `id: string`
- `ip: string`

**Example:**
```yaml
lamps:
  - type: "yeelight"
    id: "my-yeelight"
    ip: "192.168.1.100"
```

**Notes:**
- Make sure your Yeelight bulbs are on the same network as LumeHaven
- Bulbs must be in "LAN Mode" (not cloud mode)
- For Home Assistant users: Works with the [YeeLight integration](https://www.home-assistant.io/integrations/yeelight/)

##### Logging Lamp

A debugging lamp that logs lighting commands to the console instead of controlling a physical lamp.

**Required Fields:**
- `type: "logging_lamp"`
- `id: string`

**Example:**
```yaml
lamps:
  - type: "logging_lamp"
    id: "console-debug"
```

**Use Case:**
- Testing LumeHaven without physical lamps
- Debugging effect triggers
- Development and troubleshooting

### 5. Effects (`effects`)

Maps game events to lighting effects. Each effect triggers when the corresponding game event occurs.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `event` | string | Yes | Name of the game event to listen for. Must match a key in the [Event Conditions Map](#event-condition-reference) |
| `effect` | string | Yes | Type of lighting effect. Currently supports: `"pulse"` |
| `rgb` | list | Yes | RGB color as a list of three integers `[red, green, blue]` where each value is 0-255 |

**Example:**
```yaml
effects:
  - event: "fire_element_active"
    effect: "pulse"
    rgb: [255, 0, 0]
  - event: "character_healed_event"
    effect: "pulse"
    rgb: [0, 255, 0]
  - event: "monster_died"
    effect: "pulse"
    rgb: [255, 128, 0]
```

**Effect Types:**

| Effect Type | Description |
|-------------|-------------|
| `pulse` | Lamp quickly flashes the specified color |

### 6. Scenes (`scenes`)

Defines lighting scenes that can be manually activated or automatically triggered by game state.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Unique name for the scene |
| `colors` | list | Yes | List of RGB colors that lamps will cycle through. Each color is a list of three integers `[red, green, blue]` |

**Example:**
```yaml
scenes:
  - name: "tavern"
    colors:
      - [255, 140, 0]    # Orange
      - [255, 200, 100]  # Light orange
      - [200, 100, 0]    # Dark orange
  - name: "dungeon"
    colors:
      - [0, 0, 100]      # Dark blue
      - [50, 50, 150]    # Medium blue
      - [0, 50, 50]      # Teal
```

**How Scenes Work:**
- When a scene is activated, each lamp cycles through the scene's colors
- Lamps with the same scene will have different colors from the palette (distributed evenly)
- Scene changes trigger smooth transitions on compatible lamps

## Event Condition Reference

The following events are available for effect mapping:

### Element Events

| Event | Description |
|-------|-------------|
| `fire_element_active` | Fire element becomes active on the board |
| `ice_element_active` | Ice element becomes active |
| `air_element_active` | Air element becomes active |
| `earth_element_active` | Earth element becomes active |
| `light_element_active` | Light element becomes active |
| `dark_element_active` | Dark element becomes active |

### Combat Events

| Event | Description |
|-------|-------------|
| `monster_spawned` | A new monster appears |
| `monster_died` | A monster is defeated |
| `monster_received_damage` | A monster takes damage |
| `character_died` | A character is exhausted/defeated |
| `character_healed_event` | A character is healed |
| `Character_received_damage` | A character takes damage |

### Progression Events

| Event | Description |
|-------|-------------|
| `loot_found` | Loot is discovered |
| `character_gained_experience` | A character gains XP |

## Tips and Best Practices

### 1. Color Selection

- Use warm colors (reds, oranges) for combat scenes
- Use cool colors (blues, greens) for exploration scenes
- Use bright, saturated colors for important events
- Avoid pure white (#FFFFFF) as it can be harsh on the eyes

### 2. Lamp Placement

- Place lamps behind or to the side of your gaming area for best effect
- Use multiple lamps for a more immersive experience
- Consider lamp brightness - brighter lamps may need lower brightness settings

### 3. Performance

- Set `interval_ms` to 1000 (1 second) for a good balance between responsiveness and resource usage
- Higher interval values (e.g., 2000-5000) reduce CPU usage but make effects less responsive
- Lower interval values (e.g., 500) make effects more responsive but increase CPU usage

### 4. Multiple Configurations

For different campaigns or setups, maintain multiple configuration files:

```bash
# Production config
cp config.yml config-prod.yml

# Development config
cp config.yml config-dev.yml

# Run with specific config
python -m lumehaven --config config-prod.yml
```

## Troubleshooting Configuration

### Common Issues

**Lamps not responding:**
- Verify IP addresses are correct
- Check that lamps are powered on and connected to the network
- Ensure lamps are in LAN mode (not cloud mode)
- Check firewall settings

**Effects not triggering:**
- Verify `game_code` matches your GHS server
- Check that GHS server is running and accessible
- Verify SQLite database path is correct
- Check LumeHaven logs for error messages

**Scenes not changing:**
- Ensure scene names in config match API calls
- Check that `main_scene` is defined and matches a scene name
- Verify lamps are properly configured

### Validation

LumeHaven validates your configuration on startup. If there are errors, they will be logged and the server may fail to start.

Common validation errors:
- Missing required fields
- Invalid RGB values (must be 0-255)
- Unknown lamp types
- Unknown event names
- Scene referenced in `main_scene` not defined
